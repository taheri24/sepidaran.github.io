# معرفی Schema

یکی از مهم ترین موضوعات وب سرور ها کار با موجودیت Entity می باشد

  • تبدیل به JSON شدن - مسایل Serialization
  • اعتبارسنجی - Validation
  • مستندسازی در سطح API
{
    "id": "identity",
    "*userName": "string",
    "passwordHash": "string",
    "permissions": [
        {
            "id": "identity",
            "userAccountId": "number",
            "permission": "string"
        } 
    ]
}

فرمت فایل بالا نسخه اقتباسی از پروژه easy-json-schema می باشد

  • فیلد id به عنوان identity و فیلد اصلی
  • فیلد userName - با توجه به ستاره دار بودن ، اجباری است
  • فیلد passwordHash یک فیلد با نوع داده متنی می باشد
  • فیلد permissions : آرایه ای از مجوز های داده شده به کاربر می باشد - اطلاعات داخل این فیلد با جدول دیگر ارتباط یک به چند دارد

نکته

در حالت آرایه و یک به چند ، اولین فیلد داخلی مربوط identity می باشد و دومین فیلد نیز برای FK میباستی تنظیم شود
