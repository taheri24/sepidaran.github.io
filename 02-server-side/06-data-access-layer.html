# معرفی لایه دسترسی به داده

# مقدمه

منظور از لایه دسترسی به داده فایل هایی است شامل دستورات SQL میشود

  • فرمت این فایل markdown می باشد اگرچه این فرمت بیشتر برای مستندسازی می باشد - در اینجا جنبه کد دارد
  • جایگزین ORM - ما از آن استفاده نمی کنیم
  • پیاده سازی مناسب CRUD - Create Read Update Delete
  • تولید اتوماتیک قسمت CRUD با وجود امکان تغییر آن توسط برنامه نویس
  • هندل آسان داده جزئیات Master/Detial - در هنگام درج و بروزرسانی
  • داشتن قابلیت پیشرفته فیلتر دینامیک
  • امکان کش سمت مرورگر

چرا `markdown`

استفاده از فرمت markdown به عنوان لایه داده(سورس کد) امکان جداسازی و رنگبندی دستورات SQL به توسعه دهندگان می دهند در نتیجه نیاز های پروژه(دیتابیس محور) را کاملا مرتفع نمود

# نمونه از یک لایه دسترسی به داده

## Create
```sql        
INSERT INTO UserAccount([UserName],[PasswordHash]) 
VALUES(@UserName,@PasswordHash) 
```.
-- Signed Generated Code`Create` with (4080a994f8060a0853182c6abd185d23)


## Read
```sql        
SELECT [Id],[UserName],NULL AS [PasswordHash] 
FROM [UserAccount] 
WHERE (1=1)
```.
-- Signed Generated Code`Read` with (a101431bad0aa277ffa4a538b391e20d)

## Update
```sql        
UPDATE UserAccount 
SET [UserName]=@UserName,[PasswordHash]=COALESCE(@PasswordHash,[PasswordHash])
WHERE  [id]=@id 
```.

# عملیات CRUD

CRUD به عنوان چهار عملی در مدل REST می باشد و معمولا نیز برای آن در این سطح دستورات SQL درست شده است اگر شما با MS-SQL-SERVER کار کرده باشید شکل کدنویسی ## Create,## Delete,## Update بسیار شبیه به Stored Produce میباشد ولی هدف ## Read خواندن اطلاعات همراه با فیلتر دینامیک است . هسته سیستم فیلتر دینامیک تبدیل به دستورات SQL-Where میکند و آنرا با (1=1) جایگزین میکند

نکته

سیستم Code-Generator داخل هسته سیستم کد های CRUD به صورت اتوماتیک از روی Schema تولید می کند در صورتی که آنرا بخواهید تغییر بدهید، برنامه دوباره آن قسمت را تولید نخواهد کرد و کدهای شما سالم خواهد ماند ، مکانسیزم شناسایی این تغییر بوسیله روش Hash(MD5) میباشد در زیر هر قسمت کدزیر تولید میشود -- Signed Generated Code'Permissions(Delete)' with (2aae8da2ad10542b3e088645443b283c)

# عملیات CRUD بر روی Detail

در داخل Schema آرایه ها نمایانگر Detail می باشد در مثال بالا دیتا موجود در فیلد Permission به عنوان جزئیات می باشد که برای آن CRUD تعریف شده است

  • ## Permissions(Create) برای ایجاد(درج) یکی از مجوز های مربوط به کاربر
  • ## Permissions(Read) برای خواندن همه مجوز های مربوط به کاربر
  • ## Permissions(Update) برای بروزرسانی یکی از مجوز های مربوط به کاربر
  • ## Permissions(Delete) برای حذف یکی از مجوز های مربوط به کاربر

function requestHandler(req){

    const targetUser =  await userAccountDA.singleRow('FullRead', req, { userName_eq:username });
    return targetUser
}
